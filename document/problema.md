# Descripción del problema

El trabajo se basa en un sistema de ventas y pedidos utilizado por un comercio para registrar clientes, productos, categorías y pedidos. A medida que aumenta la cantidad de datos registrados, ciertos reportes comienzan a tardar más de lo esperado.

El problema a resolver no está solamente en escribir consultas SQL correctas, sino en entender cómo las ejecuta el motor de base de datos y cómo mejorar su rendimiento mediante el uso adecuado de índices y buenas prácticas de consulta.
